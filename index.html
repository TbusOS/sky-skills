Concept概念

What is a Claude Code Skill? 什么是 Claude Code Skill？

A skill is a folder with a SKILL.md file (plus optional templates, scripts, and references) that gives Claude Code domain expertise — coding standards, templates, patterns, whole workflows. When Claude reads a user prompt, it scans installed skills and automatically loads the ones whose TRIGGER keywords match. You never invoke them by name.

一个 skill 是一个文件夹，里面放一份 SKILL.md（再加可选的模板、脚本、参考资料），给 Claude Code 喂领域知识 —— 编码规范、代码模板、工作流模式。Claude 读到用户 prompt 时会扫描已安装的所有 skill，自动加载 TRIGGER 关键词匹配的那几个。你不用手动调用。

The point is folded expertise. Prompts don't scale — they get buried, repeated, forgotten. A skill is a small, reusable module you write once and forget, and Claude applies it every time it's relevant. Think of it as the difference between handing a colleague a style guide versus telling them every time remember to use our formatting rules.

关键词是 "折叠好的专业知识"。Prompt 无法规模化 —— 会被埋没、被重复、被遗忘。Skill 是一个小而可复用的模块，你写一次、塞进去就忘，之后 Claude 每次合适的时候会自动应用。就像给同事一份风格手册，而不是每次都提醒"记得按我们的规则排版"。

A skill should feel like a colleague who already knows the domain — not a style guide you have to quote from. 一个好的 skill 应该像一位已经懂行的同事 —— 而不是你需要反复引用的风格手册。

Architecture架构

Generators write. Evaluators judge. 生成器写稿。评审员判稿。

Two principles taken from Anthropic's harness-design post and the GAN paradigm: (1) the agent producing work tends to praise its own work; (2) a separate, skeptical evaluator is a powerful lever. So the 4 design skills are generators. design-review is the evaluator. They live in different folders on purpose. 两条原则来自 Anthropic 的 harness-design 文章 和 GAN 范式：(1) 做事的 agent 倾向自我称赞自己的作品；(2) 独立而持怀疑的 evaluator 才是真正的杠杆。所以 4 个设计 skill 是 generator，design-review 是 evaluator，它们刻意住在不同文件夹。

Separation分离

Generator ≠ Evaluator 生成 ≠ 评审

The evaluator lives in its own skill folder with its own rules. It does not see the generator's internal reasoning — only the rendered output. That is how GAN's discriminator stays sharp. 评审员住在独立 skill 文件夹，有独立规则。它看不见生成器的内部推理 —— 只看渲染出来的东西。这是 GAN 的 discriminator 保持锐利的方式。

How it works工作原理

Auto-activated. Always in context. 自动激活。始终在场。

You write a prompt. Claude scans every installed skill's TRIGGER frontmatter. If it matches, the skill loads automatically. No manual invocation. 你写 prompt。Claude 扫描已安装 skill 的 TRIGGER 字段。匹配上就自动加载，不需要手动调用。

Install安装

Three ways. Pick your fit. 三种方式。按需选择。

Skills are just folders. You can install one at a time, copy the whole repo, or symlink for live updates. skill 就是文件夹。可以单个安装、整仓库复制、或者 symlink 实时同步。

1. Claude Code CLI (recommended) 1. Claude Code CLI（推荐）

One skill at a time. Claude Code pulls it directly from GitHub. 一次装一个，Claude Code 直接从 GitHub 拉取。 

claude install github:TbusOS/sky-skills/skills/apple-design
claude install github:TbusOS/sky-skills/skills/linux-kernel-dev
claude install github:TbusOS/sky-skills/skills/doc-to-markdown

2. Copy into your project 2. 复制到你的项目

Grab just the SKILL.md you need and drop it into your project's .claude/skills/. 只拿你需要的 SKILL.md，放到项目的 .claude/skills/ 下。 

git clone https://github.com/TbusOS/sky-skills.git
cp sky-skills/skills/anthropic-design/SKILL.md \
   your-project/.claude/skills/anthropic-design.md

3. Symlink (auto-updates) 3. Symlink（自动更新）

Symlink the whole folder. git pull brings new skills + fixes into every project automatically. 整个文件夹 symlink 过去。之后 git pull 就能把新 skill 和修复同步到所有项目。 

git clone https://github.com/TbusOS/sky-skills.git
ln -s "$(pwd)/sky-skills/skills/sage-design" \
      your-project/.claude/skills/sage-design
Auto-trigger自动触发

Claude activates each skill only when its TRIGGER keywords match your prompt. You never call one by name — you just describe the task, and the relevant skill loads itself. Claude 只在 TRIGGER 匹配上时加载对应 skill。你永远不用按名字调用 —— 只要描述任务，相关 skill 会自己加载。

Contribute贡献

Write your own skill. 写你自己的 skill。

A skill is a folder with a SKILL.md. The frontmatter lists TRIGGER keywords, and the body is the domain knowledge you want Claude to have. Templates, reference docs, and scripts all live under the same folder — self-contained, no external dependencies.

一个 skill 就是一个含 SKILL.md 的文件夹。frontmatter 里写 TRIGGER 关键词，正文写你想让 Claude 掌握的领域知识。模板、参考文档、脚本都放在同一个文件夹里 —— 自包含，没有外部依赖。

Design skills in this repo ship with three automated quality gates: verify.py checks structure, visual-audit.mjs renders the page in Playwright and flags contrast / sizing / orphan-layout issues, and screenshot.mjs captures a full-page image for human review. All three must exit 0 before a skill ships.

本仓库的设计 skill 都配备三道自动质量闸：verify.py 检查结构、visual-audit.mjs 用 Playwright 渲染页面并扫描对比度 / 尺寸 / 孤儿布局、screenshot.mjs 抓全页截图供人工 review。三道都 exit 0 才能发布。
