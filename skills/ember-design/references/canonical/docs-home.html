What the skill ships. 这个 skill 里有什么。

Reading order: tokens → typography → layouts → components. Each file earns its place; they're not decoration. 阅读顺序：tokens → 字体 → 布局 → 组件。每一份都在最终产出里起作用，不是装饰。

01 · TOKENS

design-tokens.md

CSS variables for warm-cream palette, chocolate brown, gold accent, spacing, radius. Components never hard-code hex. 暖米、巧克力棕、金色点缀、间距、圆角的 CSS 变量。组件禁止硬编码 hex。

02 · TYPE

typography.md

Fraunces (display serif with living italic) + Inter (body sans) + IBM Plex Mono (code / eyebrows). Italic discipline: pull-quotes and taglines only. Fraunces(有呼吸的 italic 衬线展示)+ Inter(正文 sans)+ IBM Plex Mono(代码 / 小标)。Italic 只用在 pull-quote 和 tagline。

From a Claude chat 在 Claude 对话里

Prefix your request with ember 风格 (or any trigger word in SKILL.md). The skill activates automatically. Run the three-gate design-review before shipping. 在请求前加 ember 风格(或 SKILL.md 里任何触发词)。Skill 自动激活。发布前跑三道 design-review 闸。

Read the trigger list查看触发词列表 
# 1) Ask in the ember voice
> ember 风格,写一张 Skypad landing 页

# 2) Verify structure
$ python3 skills/design-review/scripts/verify.py \
    my-page.html

# 3) Render + audit + screenshot
$ bin/design-review --skill=ember \
    my-page.html
In the harness在 harness 里

One of eight generators.
Evaluated by one discriminator. 八个生成器之一。
由一个判别器审核。

ember-design generates. design-review judges. The split is intentional — GAN-style. No generator evaluates its own output. ember-design 负责生成。design-review 负责判别。这个分离是有意的 —— GAN 式。没有生成器评判自己的输出。
