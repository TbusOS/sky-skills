# Canonical · eclat-design spec

> 什么让这成为一张好的 eclat 规格页。生成新 eclat spec 页**前先读**。
> 完整设计决策见 `spec.html` 末尾的 `design-review:self-diff v1` 注释块。

---

## 5 个让它成立的决策

### 1. 先给一个满屏规格"时刻",再上密集表
关键规格(芯片 "2×")用满屏 `.eclat-bignum` 单独占一屏,**之后**才上完整参数表。eclat 是 keynote 不是 datasheet —— 最重要的一个数字先给电影感气口。直接上表 = 读成 lectern。

### 2. 6 张速览卡 + 一张完整暗色表(两个高度)
`.eclat-spec-grid`(6 卡)给发布日"关键数字"节奏;`.eclat-spec-table`(暗色)给诚实的完整记录。读者用其一,小重叠比表不完整便宜。

### 3. 页头用 `.eclat-pagehead`(156px),不是 100vh hero
规格页是参考资料不是揭示;满屏 hero 会过度戏剧化、把参数埋到首屏下。100vh 的电影时刻只留给那一个芯片数字。

### 4. 暗色表 + mono flare-soft 分类头
表留在哑光舞台上,保持整页一块连续的 keynote 表面;浅色表会在暗底上戳个洞、读成贴进来的组件。分类头用 mono `--eclat-flare-soft` 系住品牌,不引入第二高光色。

### 5. 诚实优先的文案
"没有凑整的数字,没有小字星号" + 精确量值,而不是营销最高级。规格页是读者会随手验算的地方;这里精确,别处的电影宣称才站得住。

## 何时用
产品发布会的技术规格 / 参数页。**不要**用于商务汇报数据表(→ lectern)。
配色/字体/双语/标点规范同 [[launch.md]] 与 `dos-and-donts.md`。
