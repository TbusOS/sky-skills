# lectern-design — Do / Don't

会议室皮肤的品位边界。lectern 的身份 = **可信的商务汇报 deck**:浅底、衬线 gravity、结构化、数据/决议前置。它不靠惊艳,靠让读者**信**且**省时间**。下面按"破坏可信感的程度"排。

## 绝对不要(破坏身份 / 机器或 critic 会抓)

| Don't | 为什么 → 怎么做对 |
|---|---|
| 用暗调电影感 / 满屏巨字 / 聚光 hero | 那是 **eclat** 的语言。lectern 一旦戏剧化就读成营销发布,丢掉董事会该有的 gravity。标题用 `.lectern-title`(衬线、克制),不要 `.lectern-hero`。 |
| 全 sans 大标题(消费营销味) | 全 sans 标题读成 apple.com 那种消费页。大标题用衬线(Georgia / 宋体)—— "经过斟酌的文件" 的可信感。正文才用 sans。 |
| 彩虹多色相图表 / 高饱和色块 | 文档失信。按数据类型选配色:有序/连续(分层、时间)用**单色相明度梯**(深蓝 `#1d3a6e→#2f5bb0→#7d9bd0`);无关分类才多色相但**锁同一明度带**。线图实线=实际、灰虚线=计划。低饱和是 lectern 取信的方式。 |
| 裸 RAG(红黄绿)色点表状态 | 状态是读者要据以行动的**词**,不是要解码的颜色。用带标签药丸 `lectern-pill--done/prog/block`(青/琥珀/玫红)——灰度打印也读得出。董事会 deck 会被打印。 |
| 文字渐变 / 辉光 / 毛玻璃 | 这些是 eclat/glass 的装饰语言,放进商务 deck 立刻显廉价、不可信。lectern 一切扁平、克制、有真边界(hairline + 轻阴影)。 |
| 营销最高级形容词("革命性""颠覆""业界领先") | 汇报页是读者随手验算的地方,空形容词读成"在掩饰什么"。给**事实 + 数字 + 负责人 + 期限**,让结论自己站住。(中英 buzzword 同禁,见 language 规则) |
| KPI 数值字号 < 36px | 既弱了"关键数字"的视觉权重,又会被 hollow-card 检查误判为空卡。统计卡数值 ≥36px;横排 strip(议程)用 `flex` 不用 `grid`。 |
| 画布用纯黑/纯白极值 | 用 token:纸白 `--lectern-paper #f5f6f8`、墨 `--lectern-ink #16203a`、hairline `--lectern-line #e3e7ee`。纯黑边/纯白底太硬,失掉文件的温润可信。 |
| zh span 内半角 `,;:` / BEM `__` | 全角 `，；：`(**known-bugs 1.22**);类名单连字符 `lectern-kpi-val`,修饰符才 `--`。 |

## 一定要(可信感成立的前提)

| Do | 为什么 |
|---|---|
| **可导出数字从页内重算** | 中位数 / 计数 / 占比 / 日期跨度 / 版本号发布前用页内数据核一遍 —— 一处失实(宣称 median 14 天、页内算出 25 天)全页失信。**known-bugs 1.35** |
| **决议前置、可行动** | deck 存在是为了拍决定。把"待决事项"做成独立编号 section + 决议表(决定 / 负责人 / 期限 / 状态),别埋在图表堆后面。标题敢给结论("三个决定,不是三十页幻灯")。 |
| **Confidential 标在 topbar** | 内部文件就标明,设定语域也诚实(会被截图转发)。它也锚定玫红状态色,让整套配色读起来是有意的。 |
| **数据密度** | lectern 的读者(扫数字的董事)**要**密度,一屏 KPI + 两图 + 表是对的 —— 这正是和 eclat"一屏一个气口"相反的地方。别拉稀浪费读者时间。 |
| **双语对等 + 单位不重复** | 每个 `.lang-en` 必有 `.lang-zh`;数字 + 单位双语各读一遍(`48h` + "小时" 别叠)。**known-bugs 1.41** |
